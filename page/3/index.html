操作系统 实验一 操作系统接口

发表于 2018-09-19 | 分类于 操作系统 ， 上机 | 阅读次数:

实验目的

  1. 理解操作操作系统的接口设计

实验内容

  1. 熟悉 DOS 的基本命令，包括 md，cd，copy，move，del，type 等的使用

1 当前目录下建立子目录 TEMP1 和 TEMP2，将当前目录设定为 TEMP; 2 在当前目录下创建新文件 1.txt,其内容为:这是我的记事本;
3 使用 type 命令显示 1.txt 的内容，检查正确后，执行它;
4 拷贝 1.txt 到路径 MYTEMP 中;

5 删除 TEMP2 中的文件 1.txt，删除目录 TEMP2;

  1. 编写一个备份桌面所有文件到 D:\desktop 目录的批处理文件，包括 echo，attrib，xcopy 命令的使用

  1. 修改该批处理文件的属性为只读、系统、隐藏，这样该文件对于普通用户不可见。

  2. 使用 xcopy 命令在 d 盘创建一个新文件夹名为 desktopbak，然后将桌面上所有文件 及文件夹(包括子文件夹)都复制到 d:\desktopbak。

  3. 修改 d:\desktopbak 文件夹属性为只读、系统、隐藏，这样该文件对于普通用户不 可见。

  4. 批处理文件建立、编辑。使用记事本编辑后，另存为的文件名称:Desktopycopy.bat。 即创建了文件名为 Desktopycopy 批处理文件。

  1. 理解 WINDOWS 下的编程接口原理，了解 WINAPI，利用 WINAPI 实现 WINDOWS 下的打印。

  1. 登录进入 Windows

  2. 在“开始”菜单中单击“程序”-“Microsoft Visual Studio 6.0”-“Microsoft Visual C++ 6.0”命令，进入 Visual C++ 窗口。

  3. 在 File 菜单单击 New Workspace…命令，创建 PrintApp.dsw 项目文件。

  4. 在 File 菜单单击 New C++ Source file 命令，创建新的原文件。

  5. 实验运行结果中所示的源代码，调试、编译并运行。
  6. 执行结果，理解各个函数的功能。

实验准备

在 Windows 下，基本的打印过程一般有如下六步:
1 调用 CreateDC() 函数获得打印机 DC (设备描述环境) 。
2 调用 StartDoc() 函数启动文档。
3 调用 StartPage() 函数打印一页。
4 把文档提交给打印机 DC。
5 调用 EndPage() 函数结束一页。
对文档中要打印的每一页都重复步骤 3) 至 5) 。
6 调用 EndDoc() 函数结束打印任务。 为了确定与系统相连接的打印机的型号，应用程序必须搜索用户的 WIN.INI 文件或调用

Windows 的 EnumPrinters() API 函数。而调用 EnumPrinters() 函数是最容易的方法。 EnumPrinters() 函数的定义如下所示:
